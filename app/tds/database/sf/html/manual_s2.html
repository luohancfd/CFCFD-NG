MANUAL SQL SEARCH PAGE


This is the Manual Search site, where you have to key in a SQL statement in order to perform a search. "Manual Search" is not highly recommended especially for people without technical grounding. However, it is provided for users who know SQL well enough and need an alternative searching method. For detail information regarding how to perform a search and different query types, please refer to the Instruction Menu or click here for more help. For other search options, please select from the search menu bar (the ones with green buttons) above.

You can click "Home" at the top right hand corner to return to the homepage. Additionally, you can easily return back to this page by clicking "Back" provided in the Results Page.

Note: Quick Search is highly recommended!
 

INSTRUCTIONS


In order to do a search, you are required to key in a SQL query in the textbox provided. Remember to press the SUBMIT QUERY button to get the results and press the CLEAR button to clear the textbox. EASY! And if you require additional raw data to be included into your results, just simply put a tick in the box provided. Besides, you can choose the 2 different types of display format for your results by checking the box provided as well. Done!

An SQL statement should looks something like that: "SELECT ______ FROM table name WHERE ______"

The following will show you how to form a proper SQL statement:

Keywords : SELECT, FROM and WHERE
Table Name : shot_descriptions (all must in LOWERCASE and remember included a "s")
Search using : e.g. shot_number (all must in LOWERCASE)
When combined : "SELECT * FROM shot_descriptions WHERE shot_number=8000"

Note: * stands for all available information

TWO TYPES OF DISPLAY FORMAT

Type 1 : Table format where columns=fields and rows=records. This is especially useful if your results is NOT too huge.
For an example : If you typed in project, blame and shot_number, it will show:
project blame shot_number
scramjet SF/PJ 1234

Type 2 : Displays each result individually. This is especially useful if your results is real HUGE.
For an example : If you typed in *, which indicates show all fields, it will show:
Field Names Values
project scramjet
blame SF/PJ
shot_number 1234
... ...
